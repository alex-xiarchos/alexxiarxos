Data Mining Project

Report·Source Code

Data mining and machine learning system to classify and cluster human activity data. The dataset consists of sensor readings from wearable devices, containing motion data across different activities such as walking, running, cycling, and standing. The data preprocessing pipeline uses Pandas and NumPy for handling missing values, normalizing sensor readings, and structuring the dataset for analysis. Matplotlib and Seaborn are used for visualizing patterns and correlations through heatmaps, boxplots, and time-series graphs.

For classification, three machine learning models were implemented using Scikit-learn: Neural Networks (MLP) , Random Forest, and Bayesian Networks. The models were trained and evaluated using accuracy, precision, recall, and F1-score metrics. Random Forest achieved the highest accuracy but showed signs of overfitting, while Neural Networks provided the best balance between training and testing performance.

For clustering, K-Means and DBSCAN were implemented to group activity patterns. K-Means effectively separated activity groups, whereas DBSCAN struggled with high-dimensional data.

Information Retrieval Project
Python·NumPy·
Report·Source Code

Information Retrieval system built using Python, designed to process and rank text documents efficiently. The implementation includes a Vector Space Model (VSM) with TF-IDF weighting and a deep-learning-based retrieval model (ColBERT). The system performs document preprocessing, inverted indexing, and query ranking to retrieve the most relevant results.

The preprocessing pipeline removes stopwords, applies stemming, and tokenizes text using NLTK. An inverted index is created to optimize search queries, mapping terms to document occurrences for fast retrieval. The VSM model calculates TF-IDF weights using two different weighting schemes and ranks documents based on cosine similarity. The ColBERT model is integrated using Google Colab, leveraging a pretrained transformer-based architecture for contextualized ranking.

Multi-dimensional Data Structures Project
Keyword1·Keyword2
Report·Source Code

Data structures system to efficiently store, query, and compare data points in a multi-dimensional space. A custom dataset of computer scientists was built by scraping Wikipedia with BeautifulSoup and storing structured information in a CSV format.

To enable efficient spatial searching, four advanced data structures were implemented: Range Tree, K-D Tree, R-Tree, and Quad Tree. Each structure handles multi-dimensional queries, allowing fast range searches based on scientists' names, awards, and education background. The Range Tree and K-D Tree utilize balanced binary search trees (BBSTs), the R-Tree employs bounding boxes for spatial indexing, and the Quad Tree recursively divides the space for hierarchical searching.

Additionally, the project includes an approximate similarity search algorithm using Locality-Sensitive Hashing (LSH). The LSH model applies MinHashing techniques to efficiently compare educational backgrounds, using Jaccard similarity and One-Hot Encoding to represent textual information. The text preprocessing pipeline integrates tokenization, stopword removal, stemming, and regex parsing to clean and structure data.

Benchmarking showed K-D Tree and Range Tree had the fastest queries, R-Tree handled irregular data well, and Quad Tree was slower due to recursion.
